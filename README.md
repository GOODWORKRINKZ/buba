# BU04 UWB — Телеметрия дистанции и направления

Прошивка для **ESP32-C3 SuperMini** + **BU04 UWB** (DW3000 + STM32F103, две антенны).  
Платформа определяет **дистанцию** и **направление** до тега в руках пользователя.

---

## Два режима работы

### Режим PDOA (по умолчанию) — 2 устройства ⭐

```
Платформа                    Пользователь
┌───────────────────────┐
│ ESP32 ANCHOR1 (PDOA)  │      ┌─────────────┐
│ BU04 с 2 антеннами    │      │ ESP32 TAG   │
│ (дистанция + угол)    │      │ BU04 (тег)  │
└──────────┬────────────┘      └──────┬──────┘
           │ ◄─────────────────────── │
           ▼ USB Serial 115200
     PDOA,addr_hex,seq,range_m,angle_deg,x_m,y_m,ts_ms
```

BU04 имеет **две антенны** и чип DW3000 с поддержкой **PDOA** (Phase Difference
of Arrival). Один якорь измеряет и дистанцию, и угол — ANCHOR2 не нужен.
Активируется командой `AT+SETUWBMODE=1`.

### Режим TWR (альтернатива) — 3 устройства

```
Платформа                                 Пользователь
┌──────────────┐  30 см  ┌──────────────┐
│ ESP32 ANCHOR1│◄ESP-NOW►│ ESP32 ANCHOR2│      ┌─────────────┐
│ BU04 (база)  │         │ BU04 (база)  │      │ ESP32 TAG   │
└──────┬───────┘         └──────┬───────┘      │ BU04 (тег)  │
       │ d1                     │ d2            └──────┬──────┘
       │◄───────────────────────┼───────────────────►  │
       ▼ USB Serial 115200
  PLATFORM,seq,d1,d2,angle,x,y,ts_ms
```

Два BU04 на платформе (база) измеряют расстояние до тега. ANCHOR2 отправляет d2
на ANCHOR1 через **ESP-NOW** (без проводов). ANCHOR1 вычисляет координаты x/y и угол.

## Подключение BU04 → ESP32-C3

> Источник: BU04 规格书 V1.0.0, таблица 5.  
> **BU04 управляется AT-командами по UART. SPI не нужен.**

| BU04 пин | Название | ESP32-C3 | Примечание |
|----------|----------|----------|------------|
| 34 / 24 / 23 | 3V3 | 3V3 | Пик тока ≥ 500 мА! |
| 1 / 10 | GND | GND | |
| **4** (PA2) | USART2_TX | **GPIO 3 (RX)** | BU04 → ESP32 |
| **5** (PA3) | USART2_RX | **GPIO 2 (TX)** | ESP32 → BU04 |

> ⚠️ **Питание критично.** USB ESP32-C3 SuperMini обычно даёт ~250 мА. DW3000 в BU04 при пуске потребляет до 500 мА. Если питания не хватает — модуль возвращает `INIT FAILED` на любую AT-команду, и прошивка выводит:
> ```
> # АППАРАТНАЯ ОШИБКА: BU04 вернул 'INIT FAILED'
> ```
> Решение: внешний LDO/DC-DC ≥ 500 мА на 3V3 BU04, либо качественный USB-блок и короткие толстые провода.

> Если USART2 не отвечает: попробуйте PA9 (пин 26) TX и PA10 (пин 27) RX (USART1).

## Быстрый старт (PDOA, 2 модуля) ⭐

### 1. Установите PlatformIO

```bash
pip install platformio
```

### 2. Прошейте якорь и тег

```bash
pio run -e anchor1_pdoa -t upload   # якорь (платформа)
pio run -e tag          -t upload   # тег (пользователь)
pio device monitor -b 115200
```

### 3. На якоре появится

```
# BU04 UWB – ANCHOR1 (PDOA режим, 1 якорь)
# BU04 уже в режиме PDOA
# Формат вывода: JSON
# Добавьте тег: AT+ADDTAG=<LongAddr64>,<ShortAddr>,1,64,0
```

### 4. Узнайте адрес тега и зарегистрируйте его

В мониторе якоря выполните `AT+GETDLIST`, затем добавьте тег:
```
AT+ADDTAG=000000004E818834,8834,1,64,0
```

### 5. Телеметрия

```
PDOA,8834,1,1.234,42,0.12,-0.34,12345
```

## Альтернативный старт (TWR, 3 модуля)

### 1. Прошейте ANCHOR1, узнайте его MAC

```bash
pio run -e anchor1 -t upload
pio device monitor -b 115200
```

В консоли появится:
```
# MAC ANCHOR1: AA:BB:CC:DD:EE:FF
```

### 2. Впишите MAC в `include/config.h`

```c
#define ANCHOR1_MAC  {0xAA, 0xBB, 0xCC, 0xDD, 0xEE, 0xFF}
```

### 3. Прошейте ANCHOR2 и TAG

```bash
pio run -e anchor2 -t upload
pio run -e tag     -t upload
```

### 4. Откройте монитор ANCHOR1 (115200 бод)

```
# BU04 UWB – ANCHOR1 (TWR режим, главный якорь)
# MAC ANCHOR1: AA:BB:CC:DD:EE:FF
# Запуск измерений…
ANCHOR1,1,1.234,1.189,12345
PLATFORM,1,1.234,1.567,42.3,0.891,0.834,12345
```

## Формат телеметрии

### TWR режим

| Строка | Формат | Описание |
|--------|--------|----------|
| `ANCHOR1` | `ANCHOR1,seq,d1_m,avg_m,ts_ms` | Дистанция ANCHOR1→TAG |
| `ANCHOR2` | `ANCHOR2,seq,d2_m,ts_ms` | Дистанция ANCHOR2→TAG |
| `PLATFORM` | `PLATFORM,seq,d1_m,d2_m,angle_deg,x_m,y_m,ts_ms` | Позиция тега в СК платформы |
| `TAG` | `TAG,seq,d_m,avg_m,ts_ms` | Дистанция со стороны тега |

**Поля PLATFORM:**

| Поле | Описание |
|------|----------|
| `d1_m` | Дистанция ANCHOR1 → TAG (м) |
| `d2_m` | Дистанция ANCHOR2 → TAG (м) |
| `angle_deg` | Угол тега от оси ANCHOR1→ANCHOR2 (°) |
| `x_m` | Проекция вдоль оси платформы (м) |
| `y_m` | Поперечное расстояние от оси (м) |

### PDOA режим

| Строка | Формат | Описание |
|--------|--------|----------|
| `PDOA` | `PDOA,addr_hex,seq,range_m,angle_deg,x_m,y_m,ts_ms` | Дистанция + угол + позиция от якоря |

**Поля PDOA:**

| Поле | Описание |
|------|----------|
| `addr_hex` | Короткий адрес тега (16-бит, hex) |
| `seq` | Порядковый номер измерения |
| `range_m` | Дистанция ANCHOR → TAG (м) |
| `angle_deg` | Угол к тегу (°) |
| `x_m` | X-координата тега в СК якоря (м) |
| `y_m` | Y-координата тега в СК якоря (м) |
| `ts_ms` | Метка времени millis() (мс) |

## Математика направления (TWR)

```
ANCHOR1(0,0) ──── B ──── ANCHOR2(B,0)

              TAG(x, y)

x = (d1² − d2² + B²) / (2·B)
y = √(d1² − x²)
β = atan2(y, x)   ← угол от оси платформы
```

> **Ограничение TWR:** знак `y` не определён (тег может быть с любой стороны оси).  
> PDOA решает эту проблему — угол определяется напрямую через фазовую разность.

---

## Об ориентации тега

Пользователь держит тег произвольно. UWB измеряет расстояние по **времени полёта (ToF)** — ориентация антенны влияет только на уровень сигнала, но не на точность дистанции.  
PDOA-угол измеряется в плоскости двух антенн **якоря** и не зависит от ориентации тега.

## AT-команды BU04

> Источник: BU03/BU04 AT指令 V1.0.6 (файл `BU03_BU04_AT_command_cn_V1.0.6.pdf`).

### Раздел 1: общие команды

| Команда | Ответ | Описание |
|---------|-------|----------|
| `AT` | `OK` | Проверка связи |
| `AT+GETVER` | `getver software:V1.0.0,hardware:V1.0.0` | Версия прошивки и железа |
| `AT+GETCFG` | `getcfg ID:X, Role:X, CH:X, Rate:X, Group:X` | Чтение конфигурации |
| `AT+SETCFG=ID,Role,CH,1,Group` | `OK` | Запись конфигурации (**Rate всегда 1**) |
| `AT+SAVE` | `OK` + перезагрузка ~3 с | Сохранить конфиг в Flash |
| `AT+RESTART` | `OK` | Перезагрузка |
| `AT+RESTORE` | `OK` | Сброс к заводским |

**Параметры AT+SETCFG:**

| Параметр | Значения |
|----------|---------|
| ID | 0–10 |
| Role | 0 = тег (TAG), 1 = база (ANCHOR) |
| CH | 0 = CH9 (7987.2 МГц), 1 = CH5 (6489.5 МГц) |
| Rate | **всегда 1** (6.8 Мбит/с — единственный вариант) |
| Group | 0–255 (якоря настраивают в группу; теги = 0) |

> ⚠️ **AT+GETSENSOR поддерживает только BU03. BU04 акселерометра не имеет.**

### Раздел 2: TWR — измерение расстояния

| Команда | Ответ | Описание |
|---------|-------|----------|
| `AT+DISTANCE` | `distance: 1.234000` | Запрос дистанции |
| `AT+GETDEV` | параметры | Получить коэффициенты (Калман, задержка антенны и др.) |
| `AT+SETDEV=l,d,ke,Q,R,a,b,pe,pd` | `OK` | Установить коэффициенты |

### Раздел 3: выбор режима

| Команда | Описание |
|---------|----------|
| `AT+GETUWBMODE` | Запросить режим (0=TWR, 1=PDOA) |
| `AT+SETUWBMODE=0` | Переключить в режим TWR |
| `AT+SETUWBMODE=1` | Переключить в режим PDOA |

> После смены режима обязателен `AT+SAVE`.

### Раздел 4: PDOA (только якорь)

| Команда | Описание |
|---------|----------|
| `AT+GETDLIST` | Список обнаруженных тегов |
| `AT+GETKLIST` | Список сопряжённых тегов |
| `AT+ADDTAG=LongAddr,ShortAddr,MinRate,MaxRate,Mode` | Добавить тег (**MaxRate макс. = 64**) |
| `AT+DELTAG=LongAddr` | Удалить тег |
| `AT+USER_CMD=0` | Формат вывода JSON |
| `AT+USER_CMD=1` | Формат вывода HEX |
| `AT+PDOASETCFG=p1,p2,p3,p4,p5,p6,p7` | Параметры PDOA |
| `AT+PDOAGETCFG` | Получить параметры PDOA |
| `AT+PDOAOFF=X` | Коррекция угла |
| `AT+RNGOFF=X` | Коррекция расстояния |
| `AT+FILTER=1` | Включить фильтрацию |
