# BU04 UWB Positioning & Robot Following

## What This Is

Система позиционирования на основе UWB-модулей Ai-Thinker BU04 (DW3000, двойные антенны).
Два ESP32-C3 + два BU04 работают в режиме PDOA: якорь измеряет расстояние и угол до тега в реальном времени, данные поступают по USB-Serial на ПК. Конечная цель — автономное следование робота за пользователем.

## Core Value

Якорь достоверно определяет расстояние (±10 см) и угол (±15°) до тега на дистанции 1–3 м — без этого робот следовать не может.

## Requirements

### Validated

- ✓ PDOA-режим: AT+SETUWBMODE=1, парсинг `Tag_Addr:..., Range:..., Angle:...` — existing
- ✓ CSV-телеметрия `PDOA,addr,seq,range_m,angle_deg,x_m,y_m,ts_ms` по USB Serial 115200 — existing
- ✓ Сборка PlatformIO: окружения anchor1, anchor2, tag, anchor1_pdoa, test_bu04 — existing

- [ ] Задокументированная процедура сборки стенда (проводка, прошивка, порядок запуска)

## Context

Существующая прошивка (`src/main.cpp`) уже реализует PDOA-режим через окружение `anchor1_pdoa`. Модули BU04 управляются исключительно AT-командами по UART1 (ESP32-C3 пины заданы в `include/config.h`). Чип DW3000 скрыт за STM32F103 внутри модуля — прямого SPI-доступа нет.

**Известные проблемы (из codebase map):**
- Регистрация тега ручная: нужно вручную вводить `AT+ADDTAG=<LongAddr64>,...` после каждой перезагрузки BU04
- Калибровочные константы `AT+PDOAOFF` и `AT+RNGOFF` определены в config.h, но **не отправляются** при старте
- `g_seq` переполняется через ~51 с (uint8, 200 мс интервал) — не критично для стенда, но потребитель должен знать

## Constraints

- **Железо**: ESP32-C3 + BU04 с двумя антеннами × 2 — другого железа нет
- **Транспорт**: USB Serial (только проводной стенд, Wi-Fi не используется)
- **Прошивка**: Arduino framework + PlatformIO, без внешних UWB SDK
- **Точность**: PDOA-угол работает в плоскости двух антенн якоря; вне этой плоскости точность падает
- **Дальность**: BU04 PDOA документирован до ~10 м в LOS; стенд проверяет 1–3 м

|----------|-----------|---------|
