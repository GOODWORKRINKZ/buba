# Requirements: BU04 UWB PDOA Test Bench

**Defined:** 2026-05-15
**Core Value:** Якорь достоверно определяет расстояние (±10 см) и угол (±15°) до тега на дистанции 1–3 м.

---

## v1 Requirements — Test Bench (текущий milestone)

### Firmware — Прошивка

- [ ] **FW-01**: Прошивка отправляет `AT+PDOAOFF` и `AT+RNGOFF` при каждом старте, до начала стриминга (сейчас: определены в config.h, но не отправляются)
- [ ] **FW-02**: Прошивка автоматически регистрирует тег после перезагрузки: выполняет `AT+GETDLIST`, парсит Long Address, отправляет `AT+ADDTAG` без ручного ввода
- [ ] **FW-03**: Тег сохраняет конфигурацию в BU04 (`AT+SAVE`) чтобы пережить пропадание питания
- [ ] **FW-04**: Прошивка стабильно стримит телеметрию ≥ 5 Гц непрерывно ≥ 60 с без WDT reset
- [ ] **FW-05**: Serial Monitor выводит диагностику при старте: версия BU04, статус PDOA, адрес зарегистрированного тега

### Python Visualizer — Визуализатор

- [ ] **VIZ-01**: Python-скрипт открывает Serial port с `dtr=False` (иначе ESP32-C3 ребутится при коннекте)
- [ ] **VIZ-02**: Полярная диаграмма в реальном времени: r = расстояние (м), θ = угол (°), обновление ≥ 10 Гц
- [ ] **VIZ-03**: Одновременное логирование в CSV: `timestamp,range_m,angle_deg,seq` — один файл на сессию с именем `logs/YYYYMMDD_HHMMSS.csv`
- [ ] **VIZ-04**: Кольца допуска на графике: ±10 см вокруг целевых расстояний
- [ ] **VIZ-05**: Индикатор потери сигнала — если нет пакетов > 1 с, плот показывает "LOST"

### Calibration — Калибровка

- [ ] **CAL-01**: Задокументирована процедура измерения `PDOA_OFFSET_DEG`: тег на 0°, считываем среднее, записываем смещение в config.h
- [ ] **CAL-02**: Задокументирована процедура измерения `RANGE_OFFSET_CM`: тег на 1.000 м, считываем среднее, записываем смещение
- [ ] **CAL-03**: Калибровочный скрипт `tools/calibrate.py` собирает 100 замеров на известной позиции и выводит рекомендованные значения `AT+PDOAOFF` и `AT+RNGOFF`

### Accuracy Test — Проверка точности

- [ ] **ACC-01**: Сетка 3×3 точек (1 м, 2 м, 3 м × 0°, +45°, −45°): RMSE по расстоянию < 10 см
- [ ] **ACC-02**: RMSE по углу < 15° в диапазоне ±45° от центральной оси якоря
- [ ] **ACC-03**: Результаты теста сохраняются в CSV и оформляются в `docs/accuracy_test.md` с заключением pass/fail
- [ ] **ACC-04**: Задокументированы физические условия стенда: высота модулей, ориентация антенн, расстояние до металлических поверхностей

### Research — Исследование (Phase 3)

- [x] **RES-01**: Исследование определяет необходимое количество BU04 (4× всего: 3 якоря + 1 тег) и режим работы (TWR-трилатерация, не PDOA) с обоснованием из патента CN105828431A и научных статей (UWBTracker ETH, IFAC 2024, MDPI Polar Robot)
- [x] **RES-02**: Архитектура системы спроектирована: сопроцессор RP2040 с 3× UART к BU04-якорям через PIO, выход UART/I2C на робота, data-over-UWB для кнопки «следуй за мной», якоря в равностороннем треугольнике (a=30-50 см), тег = BU04 + LiPo 500 мАч + внешняя всенаправленная антенна (IPEX)
- [x] **RES-03**: Создан исследовательский документ (`.planning/research/ROBOT-FOLLOWING.md`, >1000 строк) с: 4+ аннотациями научных статей с DOI, анализом SDK data-over-UWB, ограничениями антенн BU04, аппаратной спецификацией (~$42), архитектурной диаграммой, и чёткими рекомендациями для Phase 4 (калибровка)

## v2 Requirements — Robot Following (следующий milestone, после v1)

### Robot Control

- **ROB-01**: Якорь ESP32-C3 управляет двумя колёсными моторами через H-bridge по PWM (`-DROBOT_MODE=1`)
- **ROB-02**: P-контроллер: скорость пропорциональна расстоянию минус целевая дистанция (0.5 м)
- **ROB-03**: Дифференциальное рулевое управление на основе угла
- **ROB-04**: Dead zone ±10° — нет поворота при малых отклонениях угла
- **ROB-05**: Стоп при потере сигнала > 500 мс
- **ROB-06**: Ограничитель ±60°: движение только если тег в передней полусфере якоря (вне ±60° — PDOA ненадёжен)
- **ROB-07**: Кнопка start/stop на ESP32-C3

## Out of Scope

| Feature | Причина исключения |
|---------|-------------------|
| Wi-Fi / UDP транспорт | USB достаточен для стенда; добавляет сложность без ценности |
| Веб-дашборд / браузерная визуализация | Лишняя зависимость; matplotlib достаточен |
| Несколько тегов одновременно | Усложняет v1 без ценности |
| TWR-режим (2 якоря) | PDOA с двойными антеннами даёт угол из одного устройства |
| SLAM / карта помещения | Вне цели проекта |
| IMU-фьюжн (акселерометр + UWB) | Оверинжиниринг для стенда |
| Прямой SPI к DW3000 | BU04 скрывает DW3000 за STM32F103, AT-only |
| Автономная навигация / путевые точки | Вне текущей цели |
